⚠️ PHP Not Configured

Issue: PHP is not being executed on this server. You're seeing raw PHP code instead of the application.

Quick Diagnosis

Click these links to test PHP execution:

  • Minimal PHP Test - Should show "PHP works"
  • PHP Info Test - Should show PHP configuration
  • Test without .htaccess - Debugging guide

If you see PHP code instead of the expected output, PHP is not enabled.

Solution for UBC Students

Option 1: Contact Support

Email your system administrator and request:

Subject: Enable PHP for my account

Hello,

I need PHP enabled for my public_html directory to run a database project.
My username is: [your_username]
Project location: ~/public_html/parking-dbms/

The project requires:
- PHP 7.4 or higher
- OCI8 extension for Oracle database connectivity

Thank you!

Option 2: Try Different Directory

Some servers have PHP enabled only in specific directories:

  • Try ~/public_html/cgi-bin/
  • Try ~/www/
  • Check if there's a ~/php-enabled/ directory

Option 3: Command Line Testing

SSH into the server and test PHP:

$ php -v
$ php test.php
$ which php

Temporary Workaround

While waiting for PHP to be enabled, you can:

  1. View the project documentation: README.md
  2. Browse the source code: GitHub Repository
  3. Set up a local development environment with XAMPP/WAMP

For System Administrators

To enable PHP for this user:

# For Apache with mod_php
a2enmod php7.4
systemctl restart apache2

# Add to user's .htaccess or Apache config:
<Directory /home/username/public_html>
    Options +ExecCGI
    AddHandler application/x-httpd-php .php
</Directory>
